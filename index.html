1.hexo g报错，大体格式为xxxx denpendency

看到末尾有dependency这几个字，其实就不用管它了，这类报错是因为node.js版本过高，原来的一些文件的位置迁移或者是集成了，hexo在静态部署的时候找不到依赖才产生的，不影响使用，试了试网上的解决方法，我自己是没有生效的，不过看样子好多人都能解决问题，因此贴一个外部链接。

3.hexo d的第二类报错:OpenSSL SSL_read: Connection was reset

仓库认证报错，解决办法是取消http和https认证，代码：

</>git config --global https.sslVerify "false"

</>git config --global http.sslVerify "false"

4.hexo d的第三类报错:xxxTimed out

可以通过设置代理和取消代理来解决问题，大多数时间是有效的，但是偶尔会因为网络不稳定失效，这时候建议等网络稳定了再尝试hexo d上传项目

代码:

</>git config --global https.proxy

</>git config --global --unset https.proxy

2022.2.5，21：42分，这是我写下这篇博客的时间，也是我的一个新的起点的开始，嗯，第一篇博客，也是第一个见证，见证我新征程的第一步，希望数年后的我回头看到这篇博客的时候，能够自豪的告诉自己“我已经走到这场旅途的终点，要开启下一场冒险了”。那作为开门红的这篇博客，自然就要分享一些这时候的我认为最有价值的东西，虽然就我个人来说，我是很想分享赛马娘的，但是对于这篇文章来说，有点不合适，所以我想了想，分享我觉得做人最需要拥有的两个东西：梦想和胜负欲。无关知识，而是人活短短一世，要有这种心态！

《记事本圆梦计划》－熊谷正寿

熊谷正寿写的《记事本圆梦计划》，好像是2004年的时候出的一本书，分类大概是成功学，心灵鸡汤这一类的，国内翻译之后本土化，换了个名字——《你的时间有限，别为了别人而活》。这两个标题其实都扣紧了这本书的主题，不过相对而言，原作者的命名更直观一些，因为这本书就是教导你，“人活着，要心怀梦想”，并且你要随身带着本《圣经》大小的笔记本，随时把你的灵光一闪记录下来，并不断完善自己的梦想，整个过程就好像人要前往什么地方旅游，做足了旅游计划，从网上了解当地的风土人情，旅游攻略，名胜古迹，在了解充分后，如果被这个地方深深的吸引了，那么人就会开始收集自己所需要的情报：挑选必看的几处景点，规划好旅游路线，决定交通方式，选好住宿地点，准备好旅游所需的经费，前往这个地方，梦想也是这样的，当人被自己的梦想深深触动到之后，就会想办法去收集所有能将梦想化为现实的情报，并在充分了解梦想和现实的差距、实现梦想所要拥有的素质之后，梦想会化作一种力量，让人能够充满动力的度过每一天，不断向前迈进，一点一点的向着梦想中的自己靠拢，用自己的双手弥补梦想和现实的差距，将梦想亲手化为现实（梦想照进生活）。这种感觉很难描述，不过看过《蓝色时期》的朋友可能多多少少明白一些。如果你想要心怀梦想，有目标的生活，那么不妨读读这本书，如果能在这本书中发现梦想的价值，并且找到自己的梦想，那就再好不过了。
